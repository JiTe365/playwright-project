If you want to control it with an env var (toggle)

This works well because you can set the env var from the terminal (or VS Code settings), 
without changing code each time:

import { defineConfig } from '@playwright/test';
import dotenv from 'dotenv';

dotenv.config({
  quiet: process.env.DOTENV_QUIET === 'true',
  debug: process.env.DOTENV_DEBUG === 'true',
});

export default defineConfig({
  use: {
    baseURL: process.env.BASE_URL,
  },
});


Then in PowerShell, for example:

$env:DOTENV_QUIET="true"
npx playwright test


or

$env:DOTENV_DEBUG="true"
npx playwright test


Note: this only controls dotenv’s own messages. 
The earlier “Debugger attached” messages are from VS Code Auto Attach (NODE_OPTIONS), not dotenv.
