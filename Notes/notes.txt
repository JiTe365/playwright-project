Fix (recommended): allow local scripts for your user
Open PowerShell (or the VS Code terminal if it’s PowerShell) and run:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

Close that terminal, open a new one, then try:

npm --version

If your company policy blocks that
Some managed laptops prevent changing policy. Try Process scope (only for the current terminal session):
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
npm --version

Alternative workaround: don’t use PowerShell for npm
In VS Code:

Open the terminal dropdown → choose Command Prompt or Git Bash

Then run npm --version

Quick check: see what policies apply
Get-ExecutionPolicy -List

To install cross-env once:
npm i -D cross-env

Find where dotenv.config is called (your earlier case)

Use Ctrl+Shift+F and search:

dotenv.config
